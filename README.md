# tired-agent

> *让你的电脑加班，你只管躺平。*

A self-hostable system for running interactive CLI tools (like `claude`, `aider`, `codex`) on remote machines and accessing them from anywhere via a web browser — no remote desktop, no context loss on disconnect.

## Architecture

```
                    ┌── Remote PC ──┐
                    │ tired-agent    │  PTY executor
                    │ :8444          │  no web UI
                    └───────┬───────┘
                            │ HTTP (LAN)
                    ┌───────▼───────┐
┌──────────┐        │ tired-manager │        ┌── Remote PC 2 ──┐
│  Phone / │───────→│ web portal +  │───────→│ tired-agent      │
│  Browser │ HTTPS  │ proxy + auth  │  HTTP  │ :8444            │
│  SPA     │        │ :8443         │        └──────────────────┘
└──────────┘        └───────────────┘
```

## Quick start — Agent

The easiest way to try tired-agent is to install the agent on any machine:

```bash
npm install -g @tired-agent/agent
tired-agent init              # creates ~/.tiredagent with a generated token
tired-agent start             # starts on 127.0.0.1:8444
curl http://localhost:8444/health
```

Edit `~/.tiredagent/.env` to change the port, host, or log level:

```env
CLSSW_TOKEN=<auto-generated>
HOST=0.0.0.0
PORT=8444
CLSSW_LOG_LEVEL=info
```

## Quick start — Dev (full stack from source)

```bash
git clone https://github.com/clssw1004/tired-agent.git
cd tired-agent
npm install

# Build dependencies
npm run build:protocol
npm run build:web

# Start agent
npm run dev:agent -- --port 8444 --token dev-token-12345678

# Start manager (another terminal)
npm run dev:manager -- --port 8443 --token admin-token-12345678

# Start web dev server (another terminal)
npm run dev:web
# Open http://localhost:5173
```

The Vite dev server proxies `/v1/*` to the manager at `127.0.0.1:8443`.

## Auto-registration

Eliminate manual agent setup — agents register themselves on first start:

**1. On the manager**, configure a shared secret:

```bash
echo 'CLSSW_MANAGER_REGISTER_SECRET=my-shared-secret' >> .env
```

**2. Generate a registration string** and pass it to the agent:

```bash
REG=$(echo -n '{"managerUrl":"http://manager:8443","agentName":"my-pc","registerSecret":"my-shared-secret"}' | base64 -w0)

tired-agent start --register "$REG"
```

The agent calls the manager, receives an API token, and persists it. On restart, the agent reuses its `agentKey` for dedup — no duplicate entries.

## Docker (Manager)

```bash
git clone https://github.com/clssw1004/tired-agent.git
cd tired-agent

# Set the admin token
export CLSSW_MANAGER_TOKEN=your-admin-token

# Start
docker compose up -d
# Open http://localhost:8443
```

The Docker image embeds the web SPA — one container, no extra setup.

## Features

- **Session persistence** — disconnect any time. The PTY keeps running, output is captured to disk. Reconnect and resume by byte offset.
- **Mobile-friendly** — xterm.js terminal with a native input bridge for mobile keyboards.
- **Manager-Agent isolation** — the manager is a pure proxy with no PTY access; agents run independently.
- **SSE streaming** — Server-Sent Events for live output with replay from arbitrary byte offsets.
- **Agent auto-registration** — one-time setup, zero-config re-registration with dedup (`agentKey`).
